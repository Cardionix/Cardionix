Ниже приведён подробный план системы предобработки аудиозаписей сердцебиения, а затем — пример реализации на Python.

---

## 1. Постановка задачи

При записи сердцебиения с различных смартфонов качество аудио может сильно различаться. Необходимо создать универсальную систему предобработки, которая улучшит качество записей за счёт:

- Приведения аудио к единому стандарту (единичный sampling rate, моно-канал);
- Удаления шумов, возникающих от окружающей среды или особенностей микрофона;
- Выделения нужного частотного диапазона (сердечные звуки, как правило, лежат в диапазоне примерно 20–200 Гц);
- Нормализации громкости для сопоставимости между записями;
- Дополнительной визуализации (при необходимости) для контроля промежуточных этапов.

---

## 2. Обзор предобработки и выбор методов

### Шаг 1. Загрузка и ресемплирование  
- **Цель:** Унификация аудиозаписей, независимо от оригинальной частоты дискретизации.  
- **Метод:** Использовать `librosa.load` для загрузки файла и `librosa.resample` для приведения к целевому sampling rate (например, 2000 или 4000 Гц).

### Шаг 2. Применение полосового фильтра  
- **Цель:** Оставить только полезный сигнал (сердечные тоны), удаляя низкочастотные и высокочастотные шумы.  
- **Метод:** Реализовать Butterworth-полосовой фильтр с нижней границей ~20 Гц и верхней ~200 Гц (параметры можно скорректировать в зависимости от характеристик записей).

### Шаг 3. Уменьшение шума  
- **Цель:** Снижение фонового шума, присутствующего в записях.  
- **Метод:** Применить алгоритмы шумоподавления, например, спектральное вычитание. Можно воспользоваться готовой библиотекой [noisereduce](https://pypi.org/project/noisereduce/), которая позволяет выделить шум (например, из первых 0.5 секунд записи) и затем провести шумоподавление.

### Шаг 4. Нормализация громкости  
- **Цель:** Привести уровни громкости к единому масштабу, чтобы модели не «запоминали» записи по их амплитуде.  
- **Метод:** Масштабировать сигнал так, чтобы его максимальное абсолютное значение было равно 1.

### Дополнительно:  
- **Визуализация:** Для отладки можно построить волновую форму и спектрограмму с помощью `matplotlib` и `librosa.display`.

---

## 3. Архитектура системы

1. **Загрузка и унификация:**  
   - Загрузка файла, перевод в моно, ресемплирование.

2. **Полосовой фильтр:**  
   - Применение Butterworth-фильтра для выделения диапазона 20–200 Гц.

3. **Шумоподавление:**  
   - Определение фрагмента, содержащего только шум (например, первые 0.5 секунд) и применение алгоритма шумоподавления.

4. **Нормализация:**  
   - Приведение уровня сигнала к стандарту (максимум = 1).

5. **(Опционально) Визуализация:**  
   - Построение графиков для контроля промежуточных результатов.

## 4. Реализация на Python

Ниже приведён пример кода, реализующего описанную систему:

```python
import numpy as np
import librosa
import scipy.signal as signal
import noisereduce as nr
import matplotlib.pyplot as plt
import librosa.display

def load_and_resample(file_path, target_sr=2000):
    """
    Загрузка аудио, перевод в моно и ресемплирование до target_sr.
    """
    # Загружаем аудио без изменения частоты дискретизации
    y, sr = librosa.load(file_path, sr=None, mono=True)
    # Если текущая частота не совпадает с target_sr, выполняем ресемплирование
    if sr != target_sr:
        y = librosa.resample(y, orig_sr=sr, target_sr=target_sr)
        sr = target_sr
    return y, sr

def bandpass_filter(y, sr, lowcut=20, highcut=200, order=4):
    """
    Применение полосового фильтра с использованием Butterworth-фильтра.
    """
    nyq = 0.5 * sr
    low = lowcut / nyq
    high = highcut / nyq
    b, a = signal.butter(order, [low, high], btype='band')
    y_filtered = signal.filtfilt(b, a, y)
    return y_filtered

def reduce_noise(y, sr):
    """
    Применение шумоподавления с использованием библиотеки noisereduce.
    В качестве примера выделяем первые 0.5 секунды как шумовой фрагмент.
    """
    noise_duration = 0.5  # длительность в секундах
    noise_sample = y[0:int(noise_duration * sr)]
    # reduce_noise автоматически вычисляет спектральные характеристики шума
    y_reduced = nr.reduce_noise(audio_clip=y, noise_clip=noise_sample, verbose=False)
    return y_reduced

def normalize_audio(y):
    """
    Нормализация аудио, приводящая максимум по модулю к 1.
    """
    max_val = np.max(np.abs(y))
    if max_val > 0:
        y = y / max_val
    return y

def preprocess_audio(file_path, target_sr=2000, lowcut=20, highcut=200, order=4):
    """
    Полный пайплайн предобработки аудио.
    """
    # Загрузка и ресемплирование
    y, sr = load_and_resample(file_path, target_sr)
    
    # Применение полосового фильтра
    y_filtered = bandpass_filter(y, sr, lowcut, highcut, order)
    
    # Шумоподавление
    y_denoised = reduce_noise(y_filtered, sr)
    
    # Нормализация
    y_normalized = normalize_audio(y_denoised)
    
    return y_normalized, sr

def plot_waveform(y, sr, title='Waveform'):
    """
    Визуализация волновой формы аудио.
    """
    plt.figure(figsize=(10, 4))
    librosa.display.waveshow(y, sr=sr)
    plt.title(title)
    plt.xlabel('Время (с)')
    plt.ylabel('Амплитуда')
    plt.tight_layout()
    plt.show()

def plot_spectrogram(y, sr, title='Spectrogram'):
    """
    Визуализация спектрограммы аудио.
    """
    D = librosa.stft(y)
    S_db = librosa.amplitude_to_db(np.abs(D), ref=np.max)
    plt.figure(figsize=(10, 4))
    librosa.display.specshow(S_db, sr=sr, x_axis='time', y_axis='hz', cmap='magma')
    plt.colorbar(format='%+2.0f dB')
    plt.title(title)
    plt.tight_layout()
    plt.show()

# Пример использования
if __name__ == '__main__':
    file_path = 'path_to_heart_sound.wav'  # укажите путь к аудиофайлу
    processed_audio, sr = preprocess_audio(file_path)
    
    # Визуализация результатов
    plot_waveform(processed_audio, sr, title='Processed Heart Sound Waveform')
    plot_spectrogram(processed_audio, sr, title='Processed Heart Sound Spectrogram')
```

## 5. Краткий анализ и комментарии

- **Унификация входных данных:** Ресемплирование аудио позволяет работать с разными форматами и частотами записей, что особенно важно при использовании записей с различных телефонов.  
- **Полосовой фильтр:** Фокусировка на диапазоне 20–200 Гц помогает убрать ненужные высокочастотные шумы и низкочастотные вибрации, не относящиеся к сердечным тонам.  
- **Шумоподавление:** Использование библиотеки noisereduce позволяет автоматически оценить и убрать фоновой шум, что значительно повышает качество аудиосигнала.  
- **Нормализация:** Обеспечивает сопоставимость по амплитуде между записями, что важно для последующей классификации.

Эта система предобработки поможет значительно улучшить качество аудиозаписей сердцебиения, что будет полезно при дальнейшем анализе и классификации, особенно если записи получены с различных устройств.
