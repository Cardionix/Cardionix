## 🔷 Общая цель

Мы хотим классифицировать 10-секундные аудиофрагменты сердечных звуков (кардиофонограмм) на три класса:

* **Normal** — нормальный сердечный ритм
* **Abnormal** — патология
* **Artifact** — шум, механическое искажение, артефакты записи

---

## 🔷 Проблема с аудио

Аудио — это одномерный сигнал (вектор амплитуд по времени), но:

* такие сигналы плохо воспринимаются CNN и Transformer-моделями напрямую,
* человеческие уши тоже не воспринимают звук «как есть», а анализируют **частотный состав** (через ухо → улитку → мел-спектр),
* нам нужно извлечь **времено-частотные признаки**, чтобы различить ритм, частоту, шумы.

---

## 🔷 Решение: три спектрограммы как «каналы изображения»

Мы преобразуем каждый аудиосигнал в **три разных спектрограммы**:

1. **STFT** — классическая спектрограмма (амплитуда частот во времени)
2. **Mel-спектрограмма** — приближает восприятие человека (частоты проецируются на нелинейную шкалу)
3. **Вейвлет-преобразование (CWT)** — даёт адаптивную шкалу: высокая временная точность на высоких частотах, и наоборот

Эти три представления:

* дают **взаимодополняющие признаки**
* конкатенируются в один тензор формы `[3, 128, 128]` (как RGB-изображение)

---

## 🔷 Зачем вейвлет-преобразование?

Вейвлет-преобразование (Continuous Wavelet Transform, CWT):

* работает не через фиксированные окна (как STFT), а **через масштабируемые волны**,
* даёт **лучшее разрешение по времени для высоких частот** (например, шумов, артефактов),
* **масштаб (scale)** — обратен частоте: чем выше scale → тем ниже частота.

Вместо частот, мы оперируем **scale-параметрами**, которые подбираются автоматически:

```python
scales = fc * sr / freqs
```

где `fc` — центральная частота вейвлета (Morlet), `sr` — sample rate, `freqs` — линейное разбиение от 20 Гц до Nyquist частоты.

---

## 🔷 Архитектура модели

1. **Вход:** `[3, 128, 128]` — 3-канальная карта (STFT, Mel, WT)
2. **Backbone:** `ResNet18` — извлекает spatial features → `[B, 512, H', W']`
3. **Flatten:** spatial признаки разворачиваются в последовательность токенов
4. **TransformerEncoder:** обрабатывает эти токены как последовательность (учитывая взаимосвязи между частотными областями)
5. **Head:** полносвязная сеть классифицирует на один из трёх классов

---

## 🔷 Почему это мощный подход

| Компонент   | Почему он важен                                                                                      |
| ----------- | ---------------------------------------------------------------------------------------------------- |
| STFT        | Улавливает основную частотную структуру сердечного ритма                                             |
| Mel         | Подгоняет спектр под человеческое восприятие (важно для «музыкальности» ритма)                       |
| WT          | Улавливает **резкие выбросы**, шумы, артефакты — идеально подходит для поиска нарушений и артефактов |
| CNN         | Извлекает локальные spatial признаки                                                                 |
| Transformer | Улавливает **глобальные зависимости** между частотными компонентами во времени                       |
| Resize      | Даёт единый размер всех входов, без чего batching и склейка невозможны                               |

---

## 🔷 Почему именно Morlet Wavelet

Мы используем **Morlet wavelet**, потому что:

* это гауссиана, модулированная синусоидой,
* она хороша для анализа **псевдосинусоидальных структур** (сердечный ритм — как раз такой),
* и она даёт **непрерывное представление**, хорошо подходящее для визуализации и подачи в CNN.

## 🔷 Финальный вывод

Ты строишь аудиопайплайн, который:

1. Конвертирует аудио в три согласованных спектральных представления
2. Объединяет их в 3-канальное «изображение»
3. Передаёт это через комбинацию `ResNet` (локальные признаки) + `Transformer` (глобальные контексты)
4. Получаешь компактное, интерпретируемое представление аудио
5. И получаешь классификацию с максимальным учётом структуры сигнала
